요청 :
클라이언트 -> 컨트롤러 -> 서비스 -> 레포지토리 -> 서버
응답 : 요청의 역순

과정 :
클라이언트 : 프론트엔드
컨트롤러 : 방향을 잡아줌 ( get , post , put , delete등 ... )
서비스 : 컨트롤러가 정해준 방향의 기능을 구현해줌
레포지토리 : 서비스 기능들이 db에 이동할수 있도록 해줌 (저장,반환 둘다)
서버 : mySQL ...



Client: 시스템에 데이터를 요청

Controller: Client 요청을 해석해서 요청을 Service 로 전달

Service: 비즈니스 로직을 처리하며, Controller로부터 받은 데이터를 가공하고 필요한 작업을 수행.
** 비즈니스 로직 : 시스템의 핵심 로직 (데이터 검증 , 계산 , 규칙 적용 , 데이터 가공 , 의사 결정 ... )

Repository: 데이터 액세스 계층으로, DataBase(데이터 저장소)와 상호작용하여 데이터를 읽고 쓴다.
**데이터 액세스 계층 : 데이터의 저장,검색,업데이트,삭제 ... 로직,프로세서를 처리하는부분

Server: Client의 요청을 처리하고, 서비스를 제공하는 서버 컴퓨터 또는 시스템.
